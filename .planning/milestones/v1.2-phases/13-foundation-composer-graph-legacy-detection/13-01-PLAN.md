  <action>
    Create src/Core/PSR4Resolver.mjs as a new ESM module. Export a single named class PSR4Resolver.

    The class has one public method: resolve(fqcn, projectRoot, psr4Map).

    Algorithm — extracted verbatim from PhpParser._resolvePsr4() (read that method first):
    1. Split fqcn on "\\" to get nsParts array.
    2. Iterate Object.entries(psr4Map). For each [prefix, dirs]:
       a. Strip trailing backslashes from prefix using .replace(/\\+$/, "").
       b. Split stripped prefix on "\\" to get prefixParts.
       c. Check that every prefixParts[i] === nsParts[i]. If mismatch, continue.
       d. Normalize dirs to array: Array.isArray(dirs) ? dirs : [dirs].
       e. For each dir: build relPath = join(dir, ...nsParts.slice(prefixParts.length)) + ".php"; build absPath = join(projectRoot, relPath); if existsSync(absPath) return absPath.
    3. Return null if no match found.

    Imports: existsSync from "node:fs", join from "node:path". No other imports.

    This is a pure utility — no constructor state needed. The method is stateless; psr4Map is passed per-call so Phase 14 can pass any map without instantiation overhead.
  </action>

    <automated>node --test test/composer-parser.test.mjs 2>&1 | grep -E "(pass|fail|PSR4)" | head -20</automated>
  </verify>

    - resolve returns null when no psr4Map prefix matches the FQCN

  <action>
    Create src/Core/ComposerParser.mjs as a new ESM module. Export named class ComposerParser.

    The class has one public method: parse(projectRoot).

    Internal helper readJsonSafe(filePath, label): uses existsSync — if absent, process.stderr.write warning line "# agentmap: {label} not found at {filePath}\n" and return null. Wrap JSON.parse(readFileSync(filePath, "utf8")) in try-catch — on error, process.stderr.write "# agentmap: {label} is corrupt or invalid JSON: {e.message}\n" and return null. This helper is module-private (not exported).

    parse(projectRoot) implementation:
    1. Resolve jsonPath = join(projectRoot, "composer.json") and lockPath = join(projectRoot, "composer.lock") using COMPOSER_FILES[0] and COMPOSER_FILES[1].
    2. Call readJsonSafe for both. Either or both may be null — handle independently.
    3. Build packages array. LINK_TYPES = ["require", "require-dev", "conflict", "replace", "provide", "suggest"]. If json is non-null: for each linkType in LINK_TYPES, for each [pkg, constraint] in Object.entries(json[linkType] ?? {}), push {from: json.name ?? "(root)", to: pkg, type: linkType, constraint, resolvedVersion: null}.
    4. Build resolved map from lock: merge lock.packages ?? [] and lock["packages-dev"] ?? [] into a Map of name → version string.
    5. For each entry in packages, set resolvedVersion = resolved.get(entry.to) ?? null.
    6. Extract psr4Map: merge json?.autoload?.["psr-4"] ?? {} with json?.["autoload-dev"]?.["psr-4"] ?? {} — dev namespaces also needed by PSR4Resolver for test file resolution in Phase 14.
    7. Extract classmaps: [...(json?.autoload?.classmap ?? []), ...(json?.["autoload-dev"]?.classmap ?? [])].
    8. Extract autoFiles: [...(json?.autoload?.files ?? []), ...(json?.["autoload-dev"]?.files ?? [])].
    9. Return {packages, psr4Map, classmaps, autoFiles}.

    Imports: readFileSync, existsSync from "node:fs"; join from "node:path"; COMPOSER_FILES from "./constants.mjs".

    Do NOT include vendor/ filtering here — vendor/ packages appear in lock.packages but the packages array only records root composer.json dependency declarations (which are never vendor internal entries). Root composer.json "require" lists package names like "laravel/framework", not vendor paths. No special filter needed.
  </action>

## Trust Boundaries

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | ComposerParser readJsonSafe | mitigate | JSON.parse in try-catch; never eval(); never exec subprocess; malicious keys are stored as strings only |
| T-13-02 | DoS | JSON.parse on deeply nested composer.json | accept | V8 JSON.parse has no depth limit but this is a local CLI tool; no untrusted network input; low risk |
| T-13-03 | Tampering | PSR4Resolver path construction | mitigate | Only join(projectRoot, relPath) — relPath built from composer.json psr-4 map values; paths stay within projectRoot by construction (relative paths only in spec); no traversal via ".." since psr-4 dirs are typically "app/", "src/", etc. |
| T-13-SC | Tampering | npm/pip/cargo installs | accept | No external packages installed in this phase; all node:fs and node:path built-ins |
</threat_model>
