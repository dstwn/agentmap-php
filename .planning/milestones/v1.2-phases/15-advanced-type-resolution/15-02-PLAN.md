**Add _peelChain(node):**
  Walks left-recursively via child(0) while node.type === "member_call_expression".
  Uses unshift() to build steps in left-to-right call order.
  Each step: `{ method: cur.child(2)?.text ?? "?" }`.
  Returns `{ rootObject: cur, steps }` where cur is the terminal non-member_call_expression node (the root variable).
  See RESEARCH.md Pattern 1 for the verified implementation sketch. Follow it exactly — the child index mapping (0=object, 2=name) is confirmed via live AST dump.

**Add _findMethodReturnType(root, methodName):**
  Walks the AST for method_declaration nodes where the name child text === methodName.
  On match, scans the method_declaration's children for return type nodes.
  Handles all four node types per RESEARCH.md Pattern 2:
    - named_type → return .text with leading backslash stripped
    - primitive_type → return .text (scalar; caller will treat as unknown)
    - optional_type → extract inner named_type child via _findChild; strip leading backslash
    - union_type → split .text on "|", strip leading backslashes, find first non-null/non-void/non-scalar entry
  Handle "static" and "self" returns: return the string as-is; _walkChain resolves these to currentClass.
  Returns null if no return type declared or method not found.

**Add _walkChain(currentClass, steps, useMap, psr4Map, projectRoot, depthLimit, visited, fileCache, depth):**
  fileCache defaults to new Map() if not supplied. depth defaults to 0.
  Iterates steps array:
    1. If depth >= depthLimit: write warning to stderr — `process.stderr.write(\`# agentmap: chain depth limit (\${depthLimit}) reached at \${currentClass}::\${step.method}\\n\`)` — then break.
    2. Build key = `\${currentClass}::\${step.method}`. If visited.has(key) → break. Else visited.add(key).
    3. Resolve FQCN: `const fqcn = useMap[currentClass] || currentClass`.
    4. If psr4Map empty (Object.keys(psr4Map).length === 0) → break.
    5. Call `new PSR4Resolver().resolve(fqcn, projectRoot, psr4Map)` → filePath. If null → break.
    6. Check fileCache for filePath. If miss: readFileSync(filePath, "utf8") inside try-catch (break on error), parse via `this._parser.parse(text).rootNode`, store in fileCache.
    7. Call _findMethodReturnType(classRoot, step.method) → returnType.
    8. Resolve "static"/"self" to currentClass.
    9. Push `{ method: step.method, class: currentClass, returnType: resolvedType ?? null }` to results.
    10. If resolvedType is null → break.
    11. Set currentClass = resolvedType. Increment depth.
  Return results array.

**Add resolveChain(root, useMap, assignedTypes, psr4Map, projectRoot, depthLimit):**
  depthLimit defaults to DEFAULT_CHAIN_DEPTH imported from constants.mjs. Add import at top of file: `import { DEFAULT_CHAIN_DEPTH } from "./constants.mjs";`
  Walks root for assignment_expression nodes where RHS type === "member_call_expression".
  For each match: extracts lhs.text as variable, calls _peelChain(rhs), looks up rootObject.text in assignedTypes to get rootType (className field from assigned entries only). If rootType not found → skip.
  Creates per-variable visited Set and fileCache Map.
  Calls _walkChain(rootType, steps, useMap, psr4Map, projectRoot, depthLimit, visited, fileCache, 0).
  If chain results array is non-empty, pushes:
    `{ variable, chain, resolvedType: chain[chain.length-1].returnType, confidence: "LOW", source: "chain" }`
  Returns results array.

**Important:** readFileSync import — add `import { readFileSync } from "node:fs";` at top if not already present. Check existing imports first.

| composer.json → ComposerParser | psr4Map sourced from local filesystem; no network calls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-02 | Tampering | readFileSync in _walkChain | mitigate | Only reads paths returned by PSR4Resolver.resolve() which uses existsSync before returning; no user-controlled path input |
| T-15-03 | Denial of Service | chain depth in resolveChain() | mitigate | DEFAULT_CHAIN_DEPTH = 3 hard cap + visited Set cycle detection prevent runaway; depth limit enforced before PSR4 lookup |
| T-15-04 | Tampering | malformed PHP in chain class files | mitigate | try-catch in _walkChain around readFileSync+parse; chain stops gracefully on any error |
| T-15-05 | Tampering | ComposerParser instantiation | accept | Falls back to psr4Map = {} on any error; no data loss, chain resolution degrades gracefully |
| T-15-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
