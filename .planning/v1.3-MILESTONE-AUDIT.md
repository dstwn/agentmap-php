## Cross-Phase Integration

Three integration chains verified end-to-end against source files.

### Chain 1 — Phase 17 glob fix enables Phase 18 test file pickup

**Claim:** Phase 17's `npm test` fix causes `test/integration-laravel.test.mjs` (added in Phase 18) to be automatically discovered.

**Verified:**
- `package.json` line 22: `"test": "node --test test/*.test.mjs test/**/*.test.mjs"`
- `test/**/*.test.mjs` glob matches `test/integration-laravel.test.mjs` (file in `test/` root, depth 1)
- `ci.yml` line 38: `run: npm test` — delegates to the two-glob script on all three matrix legs
- Phase 17 verification confirmed 256 tests passing before Phase 18 file was added; Phase 18 verification confirmed 260 tests passing after

**Status:** WIRED ✓

### Chain 2 — Phase 18 clone step and Phase 19 c8 wrapper both conditioned on node-20

**Claim:** The CI fixture clone (Phase 18) and the c8 coverage step (Phase 19) are both gated on `matrix.node-version == 20`, ensuring they compose correctly on the same leg without interfering on other legs.

**Verified:**
- `ci.yml` line 33: `if: matrix.node-version == 20` (clone step)
- `ci.yml` line 44: `if: matrix.node-version == 20` (coverage report step)
- Execution order on node-20 leg: Install deps → Clone fixture (line 32–34) → Run tests (line 37–38) → Coverage report (line 43–45) → Upload artifact (line 47–53)
- Clone precedes both `Run tests` and `Coverage report` — fixture is present when integration tests execute under c8
- Node 18 and 22 legs: no clone step → `existsSync(LARAVEL)` returns false → `t.skip()` fires on all three fixture-dependent tests → no failure

**Status:** WIRED ✓

---

### Chain 3 — Phase 19 c8 wraps `npm test` which includes Phase 18 integration tests

**Claim:** `npx c8 ... npm test` re-runs the full suite including `test/integration-laravel.test.mjs`, so integration test coverage is captured in the report.

**Verified:**
- `ci.yml` line 45: `run: npx c8 --reporter=text --reporter=lcov npm test`
- `npm test` expands to `node --test test/*.test.mjs test/**/*.test.mjs` — both globs include the integration file
- c8 sets `NODE_V8_COVERAGE` which propagates to the `agentmap.mjs` subprocess so `src/Core/` modules appear in the report
- Phase 19 smoke test confirmed: 260 pass (not 256 — integration tests included), coverage table shows `src/Core/` at 88.64% stmts

**Note:** c8 runs `npm test` a second time (after the `Run tests` step). On node-20 this means the integration tests execute twice — once for pass/fail, once for coverage. The fixture is cloned once and reused. This is correct and intentional.

**Status:** WIRED ✓
