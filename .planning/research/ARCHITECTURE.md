```
.github/
└── workflows/
    └── ci.yml                    # MODIFIED: glob fix + coverage step

test/
├── helpers.mjs                   # unchanged — makeRepo/run/gitInit/cleanup
├── fixtures/                     # existing PHP/TS inline fixture objects
├── vue-sfc/                      # existing (now included by glob fix)
│   └── *.test.mjs
│
├── integration-laravel.test.mjs  # NEW — integration tests vs real fixture
│
└── *.test.mjs                    # existing unit/black-box tests

coverage/                         # gitignored — c8 output dir
└── lcov.info                     # generated by c8 in CI
```

### Structure Rationale

- `test/integration-laravel.test.mjs` lives in `test/` root so existing glob `test/*.test.mjs` catches it. No new subdirectory needed.
- Coverage output in `coverage/` matches c8 defaults and what `codecov/coveralls` GitHub Actions expect.
- The laravel/framework clone lives in `tmp/eval/laravel-framework/` (shared with eval/bench harnesses) — integration tests reuse it or skip if absent.

## Architectural Patterns

### Pattern 1: Subprocess Black-Box Testing (existing, extended)

**What:** Tests invoke `node agentmap.mjs <flags>` via `execFileSync` in a tmp repo. Assert `{ stdout, stderr, status }`.
**When to use:** All agentmap CLI tests. Integration tests use the same pattern but against the real `laravel/framework` clone instead of a synthetic `makeRepo()` fixture.
**Trade-offs:** Zero coupling to internals; slow (one node process per assertion); no mocking needed.

**Example for integration test:**
```js
import { test } from "node:test";
import assert from "node:assert/strict";
import { run } from "./helpers.mjs";
import { existsSync } from "node:fs";
import { join } from "node:path";

const LARAVEL = join(import.meta.dirname, "..", "tmp", "eval", "laravel-framework");
const SKIP = !existsSync(LARAVEL);

test("--packages lists laravel/framework deps", { skip: SKIP }, () => {
  const r = run(LARAVEL, "--packages");
  assert.equal(r.status, 0);
  assert.match(r.stdout, /symfony\//);
});

test("--types lists PHP type info", { skip: SKIP }, () => {
  const r = run(LARAVEL, "--types");
  assert.equal(r.status, 0);
});

test("--legacy exits 0", { skip: SKIP }, () => {
  const r = run(LARAVEL, "--legacy");
  assert.equal(r.status, 0);
});
```

### Pattern 2: c8 Coverage Wrapping (new for CI)

**What:** `c8` sets `NODE_V8_COVERAGE` before running `node --test`. V8 writes per-process JSON coverage files for every spawned process — including the `agentmap.mjs` subprocess. c8 merges all JSON files and emits lcov/text.
**When to use:** CI coverage step only. Local dev can use `npm test` without c8.
**Trade-offs:** Adds one dev dependency (`c8`). Subprocess coverage collection requires `NODE_V8_COVERAGE` propagation — c8 handles this automatically. Does NOT require modifying test files.

**CI step:**
```yaml
- name: Coverage
  run: npx c8 --reporter=lcov --reporter=text-summary node --test test/*.test.mjs test/**/*.test.mjs
```

### Pattern 3: Conditional Skip on Missing Fixture (new pattern for integration)

**What:** Integration tests check `existsSync(LARAVEL_DIR)` and pass `{ skip: true }` if clone is absent.
**When to use:** Any test that requires a large network-fetched fixture (laravel/framework).
**Trade-offs:** CI must clone the fixture before running tests, or the integration tests silently skip. Use a dedicated CI step to ensure the clone exists.

**CI fixture step:**
```yaml
- name: Clone laravel/framework fixture
  run: |
    mkdir -p tmp/eval
    if [ ! -d tmp/eval/laravel-framework ]; then
      git clone --depth 1 https://github.com/laravel/framework tmp/eval/laravel-framework
    fi
```

## Data Flow

### Integration Test Flow

```
CI step: clone laravel/framework → tmp/eval/laravel-framework/
    ↓
node --test test/integration-laravel.test.mjs
    ↓
run(LARAVEL, "--packages") → execFileSync(node, [agentmap.mjs, "--packages"], {cwd: LARAVEL})
    ↓
agentmap.mjs reads composer.json → outputs package graph
    ↓
test asserts stdout contains expected packages (symfony/, doctrine/, etc.)
```

### Coverage Data Flow

```
c8 sets NODE_V8_COVERAGE=./coverage/tmp
    ↓
node --test forks test files (each as subprocess)
    ↓
each test file forks agentmap.mjs via execFileSync
    ↓
V8 writes coverage JSON for: node --test process + each test file + each agentmap.mjs invocation
    ↓
c8 reads all JSON files from ./coverage/tmp
    ↓
c8 merges + remaps → lcov.info, text-summary to stdout
    ↓
GitHub Actions uploads lcov.info (optional: codecov-action)
```

### Key Data Flows

1. **Glob fix flow:** CI `run:` line gains `test/**/*.test.mjs` → shell expands to include `test/vue-sfc/*.test.mjs` → 43 previously skipped tests now run.
2. **Coverage subprocess flow:** `NODE_V8_COVERAGE` env var propagates through `execFileSync` into `agentmap.mjs` child process → V8 writes coverage for the CLI code, not just the test harness.

## Scaling Considerations

This is a developer tooling test pipeline — scaling means "as test count grows":

| Scale | Architecture Adjustments |
|-------|--------------------------|
| Current (256 tests) | Single `node --test` invocation is fine |
| +Integration (256 + ~10 new) | Same invocation, integration tests skip in < 1s if fixture absent |
| Large fixture matrix | Add `--test-concurrency` flag; laravel clone cached between runs |

## Anti-Patterns

### Anti-Pattern 1: Using `--experimental-test-coverage` for subprocess coverage

**What people do:** Add `--experimental-test-coverage` to the existing `node --test` run line.
**Why it's wrong:** Only covers files *imported* by the test process. `agentmap.mjs` is spawned as a completely separate process via `execFileSync` — its lines never appear in the coverage report. You'd get coverage of `test/helpers.mjs` but 0% of `agentmap.mjs`.
**Do this instead:** Use `c8` to wrap the entire `node --test` invocation. c8 sets `NODE_V8_COVERAGE` which V8 honours in every spawned subprocess, including the agentmap.mjs child processes.

### Anti-Pattern 2: Importing agentmap.mjs in integration tests

**What people do:** `import agentmap from '../agentmap.mjs'` to call internals directly for "faster" integration tests.
**Why it's wrong:** Breaks the black-box contract established by all existing tests. Couples tests to internal API surface. Also defeats the point — the requirement is to test real CLI flags.
**Do this instead:** Keep `run(dir, "--packages")` pattern. The subprocess overhead is acceptable; the isolation is the value.

### Anti-Pattern 3: Hardcoding fixture assertions against specific laravel/framework output

**What people do:** Assert exact line counts or specific class names from the current laravel/framework HEAD.
**Why it's wrong:** laravel/framework changes upstream. Pinned assertions rot. CI breaks on `--depth 1` clone of a new commit.
**Do this instead:** Assert structural invariants — exit code 0, output is non-empty, contains known stable package names (`symfony/`, `doctrine/`), output is valid JSON when `--json` flag used. Not line counts or specific class names.

### Anti-Pattern 4: Running integration tests that require network in the matrix without caching

**What people do:** Clone laravel/framework inside each of the 3 matrix jobs (node 18/20/22).
**Why it's wrong:** Triples clone time and bandwidth; flaky on GitHub's network limits.
**Do this instead:** Scope integration test step with `if: matrix.node-version == 20` or use `actions/cache` keyed on the laravel/framework HEAD SHA.
